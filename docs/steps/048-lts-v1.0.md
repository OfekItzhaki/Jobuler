# Step 048 — LTS v1.0 Baseline

## Phase
LTS Milestone

## Purpose

This step marks the current state of the Jobuler codebase as the **LTS v1.0 baseline** — a stable, fully-tested version that all future improvements build on.

## What is working at this baseline

### Backend (ASP.NET Core API on port 5000)
- Authentication: login, register, forgot password, reset password, refresh tokens
- Spaces: list, create
- Groups: list, create, soft-delete, restore, rename, ownership transfer
- Group members: add by email/phone, remove, list with phone numbers
- Group alerts: create, list, delete (admin-only)
- Group messages: create, list, delete
- Task types: create, list
- Task slots: create, list
- Constraints: create, list
- People: list with roles and groups
- Notifications: list, mark read
- Schedule versions: list, get current (404 when none published)
- Forgot/reset password via INotificationSender (NoOp logs token to console)

### Frontend (Next.js on port 3000)
- Login, register, forgot-password, reset-password pages
- Sidebar: המשימות שלי, הקבוצות שלי
- Groups list page with avatars
- Group detail page with tabs:
  - סידור (schedule with day/week toggle, date navigation)
  - חברים (members with phone numbers)
  - התראות (alerts with severity badges)
  - הודעות (group messages)
  - משימות (task types + scheduled tasks, admin only)
  - אילוצים (constraints with user-friendly form, admin only)
  - הגדרות (rename, solver horizon, soft-delete, ownership transfer, admin only)
- My Missions page with search

### Database
- PostgreSQL 18 running locally as Windows service
- All 13 migrations applied (000–013)
- Migration tracking via `schema_migrations` table
- `jobuler` user has full permissions on all tables
- Seed data: admin, ofek, yael, viewer, dana users with correct UUIDs and linked people

### Test coverage
- 21/21 API endpoint tests passing (`infra/scripts/test_all_features.ps1`)
- Backend property tests: GroupAlertPropertyTests, PasswordResetPropertyTests
- Frontend property tests: phoneNumberRendering, alertSeverity, alertDeleteButtons, groupAvatar, transferDropdown

## Known limitations at this baseline (to be addressed in next iteration)
- No edit/delete for constraints, task types, task slots
- No admin delete for any alert (only own alerts)
- No admin delete/pin for messages
- No scheduling activation UI (solver exists in backend but no UI trigger)

## How to run

```bash
# API (port 5000)
cd apps/api
dotnet run --project Jobuler.Api/Jobuler.Api.csproj --no-build -c Debug

# Frontend (port 3000)
cd apps/web
npm run dev

# Run migrations (if needed)
powershell -ExecutionPolicy Bypass -File infra/scripts/migrate.ps1 -Password <postgres_password>

# Verify all features
powershell -ExecutionPolicy Bypass -File infra/scripts/test_all_features.ps1
```
