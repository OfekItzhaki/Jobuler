This is the LTS v1.1 checkpoint. All Phase 8 hardening work is complete and stable.

## Changes since LTS v1.0 (step 048)

### Bug fixes
- **Solver 400 error** — `schedule_runs.status` and `trigger_type` columns were still PostgreSQL enum types. EF Core sends text values, causing a type mismatch. Migration 020 converts both columns to TEXT with check constraints.
- **RLS session variable** — `TriggerSolverCommand` and `SolverPayloadNormalizer` now set `app.current_space_id` on the DB connection before querying, fixing intermittent 400s caused by EF Core connection pooling.
- **Solver group tasks** — `SolverPayloadNormalizer` was only reading legacy `TaskSlots`. Now also reads `GroupTasks` (the model used by the UI) and merges them into the solver payload.
- **Solver horizon** — was hardcoded to 7 days. Now reads the max `SolverHorizonDays` across all groups in the space.
- **Invitation accept** — invite URL was hardcoded to `https://jobuler.app`. Now reads from `App:FrontendBaseUrl` config. Login page now handles `?redirect=` param so the token survives the auth flow.
- **Build errors** — `LocalDiskFileStorage` used `IWebHostEnvironment` (web-only). Replaced with `IHostEnvironment`. Test build errors fixed (`CreatePersonCommandHandler` signature change, `GroupMemberDto` new fields).

### Security
- **DB errors no longer leak to clients** — `ExceptionHandlingMiddleware` was passing raw `DbUpdateException` messages to the API response. All DB errors now return a generic message; full error is logged server-side only.
- **Invitation accept** — person lookup now verifies `SpaceId` matches the invitation's space.

### New features
- **Image upload** — `POST /uploads/image` with magic-byte validation, local disk storage (`IFileStorage` abstraction, S3-ready). `<ImageUpload>` component wired into profile page and group member edit modal.
- **Notification bell** — moved from topbar to sidebar, next to the Jobuler logo/space name.

### UI fixes
- Constraint button had duplicate `+` (SVG icon + text). Removed `+` from text label.
- Content area now fills full width excluding sidebar (`width: calc(100vw - 256px)`).

### Tooling
- `infra/scripts/migrate.ps1` — Windows PowerShell migration runner (uses `postgres` superuser, handles ownership issues).
