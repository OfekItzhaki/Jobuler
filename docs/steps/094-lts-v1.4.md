# Step 094 — LTS v1.4: i18n Completion, Hydration Fixes, and UI Bug Fixes

## Phase
Phase 9 — Quality & Internationalisation (LTS Checkpoint)

## Purpose
This LTS checkpoint consolidates a full i18n pass across the web app, fixes several runtime errors introduced during that pass, resolves a hydration mismatch, and fixes two UI bugs (notification bell page refresh, kitchen headcount).

## What was built / fixed

### i18n — Hardcoded Hebrew removal (step 093)
- Removed all hardcoded Hebrew strings from every TSX component in the web app
- Added `useTranslations` to `MembersTab`, `MemberProfileModal`, `ScheduleTab`, `SettingsTab`, `MessagesTab`, `ConstraintsTab`, `QualificationsTab`, `admin/people/[personId]/page.tsx`
- Added new `personDetail` namespace (40+ keys) to all three locale files
- Extended `groups.members_tab`, `groups.settings_tab`, `groups.schedule_tab` in `he.json` and `ru.json`
- Fixed structural JSON bug in `en.json` and `ru.json` where `invitation` keys were floating outside any object
- Completed `he.json` — was missing entire namespaces (`liveStatus`, `imageUpload`, `draftModal`, `aiParser`, `constraintEditor`, `invitation`, `optOut`, `transfer`, `spaces`, `notifications`) and many keys within `auth`, `schedule`, `groups`, `profile`, `admin`, `errors`, `common`

### Runtime error fixes
- **`groups.loading` MISSING_MESSAGE** — `groups/page.tsx` was using `t("loading", { ns: "common" })` which is invalid next-intl syntax. Fixed by adding `tCommon = useTranslations("common")`.
- **`useTranslations is not defined`** — `groups/[groupId]/page.tsx` used `useTranslations` without importing it. Added missing import.
- **`TAB_LABELS is not defined`** — `getTabLabels(t)` was defined but never called inside the component. Added `const TAB_LABELS = getTabLabels(tGroups)` inside `GroupDetailPage`.
- **Invalid `{ ns: "..." }` pattern** — Fixed in 5 files: `today/page.tsx`, `tomorrow/page.tsx`, `TasksTab.tsx`, `AlertsTab.tsx`, `ScheduleTable.tsx`. All replaced with separate `useTranslations` hooks.

### Hydration mismatch fix (step 093 continuation)
- `today/page.tsx` and `tomorrow/page.tsx` called `toLocaleDateString(undefined)` at component top level, causing SSR/client locale mismatch. Moved into `useState("")` + `useEffect` pattern.

### Notification bell page refresh fix
- `NotificationBell` was rendered inside `<Link href="/spaces">` in `AppShell`. Clicking the bell triggered the Link navigation. Fixed by moving `NotificationBell` outside the `<Link>`.

### 429 Too Many Requests on `/auth/me`
- `AppShell` called `getMe()` on every mount unconditionally. Since `AppShell` renders on every page, every navigation triggered a new API call. Fixed by skipping the call when `storedDisplayName` is already in the Zustand store.

### Kitchen headcount fix
- Kitchen task slots had `required_headcount = 1`, causing the solver to assign one person per slot and create two consecutive single-person shifts instead of one 2-person shift.
- Migration `033_kitchen_headcount_fix.sql` updates all kitchen task slots to `required_headcount = 2`.

## Key decisions
- `he.json` was rewritten from scratch to ensure completeness — the incremental approach had left too many gaps.
- The `invitation` namespace was properly wrapped in both `en.json` and `ru.json` (was previously floating keys outside any object, causing JSON parse errors).
- Kitchen headcount fix uses byte-safe matching (`encode(name::bytea, 'escape')`) to avoid Windows terminal encoding issues with Hebrew characters. Also includes an ASCII `'kitchen'` fallback. Safe to run on any environment.

## How to run / verify
1. Run migration: `psql $DATABASE_URL -f infra/migrations/033_kitchen_headcount_fix.sql`
2. Start the web app: `cd apps/web && npm run dev`
3. Switch to Hebrew locale — no MISSING_MESSAGE errors should appear in the console
4. Navigate to any group → all tabs render in the selected language
5. Click the notification bell — the page should NOT navigate to `/spaces`
6. Check kitchen task in the schedule — should show 2 assignees per slot
